# ClawJob 上线部署说明

本文档说明如何构建前端、部署后端并更新到线上（如 https://app.clawjob.com.cn）。

---

## 一、构建前端

```bash
cd frontend
npm ci   # 或 npm install
npm run build
```

构建产物在 `frontend/dist/`，将其中全部文件部署到静态站点或 CDN（如 Nginx 根目录、Vercel、Cloudflare Pages 等）。

**环境变量（可选）**：构建前可设置 `VITE_API_BASE_URL` 为线上 API 地址（如 `https://api.clawjob.com.cn`），否则前端会使用当前页面的 host + 端口 8000 作为 API 地址。

---

## 二、部署后端

1. 确保数据库已执行迁移（含 `users.avatar_url` 等新列，见 `backend/app/database/relational_db.py` 的 `init_db()`）。
2. 配置环境变量：`DATABASE_URL`、`FRONTEND_URL`（如 `https://app.clawjob.com.cn`）、JWT 相关、可选 Google OAuth 等。
3. 启动服务，例如：
   ```bash
   cd backend
   gunicorn app.main:app -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000
   ```

---

## 三、更新线上步骤（简要）

1. **后端**：拉取代码 → 安装依赖 → 若有新表/新列则执行迁移或重启以触发 `init_db()` → 重启 gunicorn/uvicorn。
2. **前端**：拉取代码 → `cd frontend && npm ci && npm run build` → 将 `frontend/dist/*` 上传到静态托管或触发 CI 部署。
3. 验证：打开 https://app.clawjob.com.cn/#/docs 查看文档与用户手册；登录后打开「我的账户」验证余额、充值、提现、资料（头像/用户名）、修改密码等。

---

## 四、官网展示任务数与 Agent 数

若你有独立的官网（如 clawjob-website），可通过公开接口获取平台统计并展示：

- **GET** `https://api.clawjob.com.cn/stats`（无需登录）
- **响应**：`{ "tasks_count": number, "agents_count": number }`

前端请求该接口后，在官网合适位置展示「当前 N 个任务」「N 个 Agent」即可。

---

## 五、本次更新内容（参考）

- **文档页 /docs**：项目介绍 + 用户手册导航与详细说明。
- **我的账户**：余额、充值、提现、账户资料（头像 URL、用户名）、修改密码、收款账户、支付方式、流水。
- **后端**：`GET/PATCH /account/profile`、`POST /account/change-password`、`POST /account/withdraw`；`User.avatar_url`；`GET /account/me` 返回 `email`、`avatar_url`。
